## How This Project Was Created

This project was generated by the `create_project` script.

### The `create_project` command

```bash
create_project <name> [options]
```

| Argument / Flag | Description |
|---|---|
| `name` | Project name or nested path (e.g. `my_project` or `NLP/sentiment`) |
| `--gpus IDS` | Set `CUDA_VISIBLE_DEVICES` directly (e.g. `--gpus 0,1`) |
| `--no-gpu` | Skip the GPU prompt, use all available GPUs (no restriction) |

If neither `--gpus` nor `--no-gpu` is passed, the script will:
1. Run `nvidia-smi` to detect available GPUs and list them.
2. Show any previously saved selection (stored in `~/.config/ai-research/gpu.conf`).
3. Prompt you to pick which GPUs to use.
4. Save your choice globally **and** write a `.env` file into the project.

The script also runs `git init` and creates an initial commit automatically.

### Examples

```bash
# Create a project — will prompt for GPU selection
create_project PG_PV

# Create a project with specific GPUs (no prompt)
create_project PG_PV --gpus 0,2

# Create a project, skip GPU setup entirely
create_project PG_PV --no-gpu

# Nested project path
create_project NLP/sentiment_analysis --gpus 0
```

### GPU configuration

The GPU selection is stored in two places:

| Location | Scope |
|---|---|
| `/Tmp/hassicir/ai-research/gpu.conf` | Global — shared across all projects, used as default for future projects |
| `<project>/.env` | Per-project — `CUDA_VISIBLE_DEVICES=...` written here for scripts to source |

To use the `.env` in your scripts, either:
- Source it: `source .env && python scripts/train.py`
- Or load it in Python with `python-dotenv` (already referenced in the training scripts)
- Or export it in your shell profile: `export $(cat .env | xargs)`

To change GPUs later, either edit `.env` directly or re-run `create_project`
on the same project name (existing files are never overwritten).

### GPU prompt before each run

Every script (`train.py`, `evaluate.py`, `predict.py`) will **prompt you to
select GPUs** before the run starts. This ensures you always know which GPUs
you are using. The prompt shows available GPUs with free memory and your last
selection as the default.

```bash
# Will prompt for GPU selection, then train
python scripts/train.py

# Skip prompt, use all available GPUs
python scripts/train.py --no-gpu

# Use specific GPUs, no prompt
python scripts/train.py --gpus 0,1
```

The `--no-gpu` and `--gpus` flags work the same way on all scripts:

| Flag | Behavior |
|---|---|
| *(none)* | Prompt to select GPUs (shows available + free memory) |
| `--no-gpu` | Use all available GPUs, no restriction, no prompt |
| `--gpus 0,1` | Set `CUDA_VISIBLE_DEVICES=0,1` directly, no prompt |

## Quick Start

### Step 1 — Set up the environment

```bash
cd test
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Step 2 — Add your data

Place your raw data file (e.g. `dataset.csv`) inside `data/raw/`.

### Step 3 — Configure the experiment

Edit `configs/data/default.yaml` to match your dataset:

```yaml
data:
  name: my_dataset
  file: dataset.csv            # filename inside data/raw/
  target_column: target        # the column you want to predict
  split_strategy: stratified   # random | stratified | temporal
  preprocessing:
    handle_missing: mean
    scale: standard
```

Edit `configs/model/baseline.yaml` to tune hyperparameters:

```yaml
model:
  name: mlp
  params:
    hidden_dims: [256, 128]
    dropout: 0.2
    lr: 0.001
    epochs: 100
```

### Step 4 — Preprocess data (optional)

```bash
python scripts/preprocess.py --config configs/default.yaml
# Or: make preprocess
```

Saves cleaned data to `data/processed/`.

### Step 5 — Train a model

```bash
# Default config
python scripts/train.py

# With overrides
python scripts/train.py --override experiment.seed=123 experiment.name=seed_test

# With a different model config
python scripts/train.py --override model=transformer

# Via Makefile
make train
```

Each run creates a timestamped directory:
```
outputs/experiments/default_20260317_143022/
├── config.yaml      # exact config snapshot
├── metrics.json     # all logged metrics
└── artifacts/
    ├── model.pt     # the trained PyTorch model
    └── training_history.png
```

### Step 6 — Evaluate a model

```bash
python scripts/evaluate.py \
    --run outputs/experiments/default_20260317_143022 \
    --data data/raw/test.csv

# Or with a standalone model file
python scripts/evaluate.py --model path/to/model.joblib --data test.csv --target label
```

### Step 7 — Run predictions

```bash
python scripts/predict.py \
    --run outputs/experiments/default_20260317_143022 \
    --data data/raw/new_data.csv \
    --output outputs/predictions/preds.csv
```

### Step 8 — Run tests

```bash
python -m pytest tests/ -v
# Or: make test
```

## Experiment Tracking

Every training run is automatically logged by `src/tracking/logger.py`:

- **Config snapshot** — the exact YAML config is saved so you can reproduce any run
- **Metrics** — logged per step or per epoch; stored as JSON
- **Artifacts** — the trained model (and anything else you log) is saved in the run dir

### Comparing runs

```bash
for d in outputs/experiments/*/; do
    echo "=== $(basename $d) ==="
    cat "$d/metrics.json" | python -m json.tool
done
```

### Weights & Biases integration

W&B tracking is built in but disabled by default. To activate it:

1. Set your API key in `.env`: `WANDB_API_KEY=your_key`
2. Enable it in `configs/default.yaml`:
   ```yaml
   wandb:
     enabled: true
   ```
   Or via CLI override: `--override wandb.enabled=true`

When enabled, W&B will log per-epoch train/val loss, final test metrics, and
upload the model artifact. Both the local `ExperimentLogger` and W&B run
side-by-side so you always have a local copy.

## Configuration System

Configs are YAML files in `configs/` following a hierarchical pattern:

```
configs/
├── default.yaml         # base — references model + data sub-configs
├── model/
│   ├── baseline.yaml
│   └── transformer.yaml
└── data/
    └── default.yaml
```

`default.yaml` pulls in sub-configs via the `defaults:` block:

```yaml
defaults:
  - model: baseline
  - data: default
```

Override any value from the CLI:

```bash
python scripts/train.py --override experiment.seed=99 model.params.n_estimators=500
```

## Adding Custom Models

1. Define your `nn.Module` in `src/models/architectures.py`
2. Register it in the `build_model()` factory function
3. Create a config in `configs/model/my_model.yaml`
4. Train: `python scripts/train.py --override model=my_model`

## Tips

- **Never modify `data/raw/`** — treat it as immutable.
- **`make help`** shows all available Makefile commands.
- **Pin deps** after installing: `pip freeze > requirements.txt`
- **Notebooks are for exploration** — move reusable code into `src/`.
- **Check `outputs/experiments/`** to review past runs and configs.
- **GPU selection** is in `.env` — edit it or re-run `create_project` to change.
